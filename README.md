# Board App

React와 Spring Boot를 기반으로 구현한 게시판 서비스입니다.  
세션 로그인, CSRF/XSRF 대응, 게시글/댓글 CRUD, 작성자 권한 검증, RESTful API 설계를 적용했습니다.

단순 CRUD 구현에 그치지 않고, 인증/인가 흐름, 예외 처리, API 구조, 프론트-백엔드 연동 방식을 직접 설계하고 부족한 부분을 개선하며 진행한 프로젝트입니다.

## 한 줄 소개
세션 인증, CSRF 대응, 작성자 권한 검증, RESTful API를 직접 적용한 React + Spring Boot 게시판 프로젝트입니다.

## 기술 스택
- Frontend: React, Axios
- Backend: Spring Boot, Spring Security, Spring Data JPA
- Database: PostgreSQL
- Build Tool: Gradle, npm

## 주요 기능
- 회원가입 / 로그인 / 로그아웃
- 로그인 사용자 정보 조회
- 게시글 목록 / 상세 조회 / 작성 / 수정 / 삭제
- 댓글 목록 조회 / 댓글 작성
- 작성자만 게시글 수정 및 삭제 가능
- 조회수 증가

## 핵심 구현 내용

### 1. 세션 기반 인증과 Spring Security 적용
- Spring Security를 사용해 세션 로그인 구조를 구현했습니다.
- 인증이 필요한 API와 공개 API를 분리해 접근 제어를 구성했습니다.

### 2. CSRF/XSRF 대응
- 세션 기반 인증 환경에서 발생하는 CSRF 문제를 해결하기 위해 CSRF 토큰 발급 endpoint와 Axios XSRF 설정을 연결했습니다.
- 상태 변경 요청(`POST`, `PUT`, `DELETE`)이 정상적으로 동작하도록 프론트와 백엔드를 함께 조정했습니다.

### 3. RESTful API 구조 개선
- 초기에는 `/list`, `/create`, `/update`, `/delete` 같은 동사형 경로를 사용했지만, 이를 `/api/boards`, `/api/boards/{id}` 형태의 리소스 중심 구조로 리팩터링했습니다.
- 댓글 API도 `/api/boards/{boardId}/comments` 구조로 통일해 일관성을 맞췄습니다.

### 4. 작성자 권한 검증
- 게시글 수정/삭제는 단순히 프론트 버튼 제어가 아니라, 서비스 계층에서 작성자 본인인지 검증하도록 구현했습니다.
- 권한이 없는 경우 `403 Forbidden`, 존재하지 않는 게시글은 `404 Not Found`로 응답하도록 처리했습니다.

### 5. 전역 예외 처리
- `@RestControllerAdvice`를 적용해 예외를 일관된 HTTP 상태코드로 변환했습니다.
- `400`, `401`, `403`, `404`, `500` 응답을 명확히 구분해 프론트와의 연동을 안정적으로 만들었습니다.

### 6. 댓글 기능 확장 구조 설계
- 댓글 엔티티에 부모 댓글 참조 구조를 고려해, 일반 댓글뿐 아니라 대댓글 확장이 가능한 형태로 설계했습니다.
- 현재는 기본 댓글 작성/조회 기능을 우선 구현하고, 이후 대댓글 기능으로 확장 가능하도록 설계했습니다.

## 트러블슈팅

### 1. CSRF 활성화 후 회원가입/로그인 요청이 403으로 차단되는 문제
- 원인: 세션 기반 인증에서 CSRF 토큰 없이 상태 변경 요청을 보내고 있었음
- 해결: `CookieCsrfTokenRepository`와 Axios XSRF 설정을 연동하고, 초기 토큰 발급 요청을 추가해 해결

### 2. 게시글 수정 페이지 진입 시 조회수가 증가하는 문제
- 원인: 게시글 상세 조회 API와 조회수 증가 로직이 같은 흐름에 묶여 있었음
- 해결: 게시글 조회 API와 조회수 증가 API를 분리해, 상세 페이지에서만 조회수가 증가하도록 변경

### 3. 댓글 엔티티 직접 반환 시 직렬화 문제 가능성
- 원인: JPA 엔티티의 연관관계를 그대로 반환하면 직렬화 문제가 생길 수 있음
- 해결: 댓글 목록 응답 DTO를 따로 두고 필요한 데이터만 반환하도록 설계

### 4. REST 경로 일관성 부족
- 원인: 초기에는 동사형 URL과 단수/복수 혼용 경로가 섞여 있었음
- 해결: 게시글/댓글 API를 리소스 중심 경로로 재정리
